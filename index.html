Triple Pendulum Simulation

This project is a test of a triple pendulum physics simulation. The animated GIF on the right shows the motion generated by numerically solving the equations of motion.

The model uses Lagrangian mechanics: the code derives the equations from L = T - V, where T is kinetic energy and V is potential energy. That gives a compact way to handle the coupled motion of three links connected by rigid rods.

How the physics works

This is not a simple Newton-force diagram solved one mass at a time. For a triple pendulum, the three angles influence each other strongly, so the equations are tightly coupled.

Using Lagrangian mechanics is convenient because it works directly with generalized coordinates such as the angles θ1, θ2, and θ3. It avoids having to explicitly solve for constraint forces in the rods.

The result is a 6-variable first-order system: three angles and three angular velocities.

How it is computed

  • The code builds a mass matrix from the Lagrangian equations.
  • It computes gravity and coupling terms from the current state.
  • It solves for angular accelerations at each time step.
  • It integrates the system with scipy.integrate.solve_ivp using RK45.
  • The resulting motion is rendered into an animated GIF with Matplotlib.

Why this approach

Newton’s laws and Lagrangian mechanics describe the same physics, but Lagrangian mechanics is usually easier for constrained multi-body systems like pendulums.

It is especially useful here because the rods are rigid and massless, and the bobs are coupled through the angles rather than through simple independent motion.

Implementation notes

The script uses numpy for the math, scipy for ODE solving, matplotlib for rendering, and pillow to write the GIF.
